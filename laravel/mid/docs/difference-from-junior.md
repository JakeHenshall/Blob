# How this version differs from the Junior version

This mid-level version keeps the same app domain but introduces stronger application structure and more realistic behaviour.

Compared with the junior version, this version adds:

- role-based access control
- authorization through Policies
- Action classes for key workflows
- notifications
- queued jobs
- activity logging
- soft deletes
- richer dashboards
- stronger testing strategy
- clearer architectural documentation

The junior version proves framework fundamentals.
This mid version proves maintainability, separation of concerns, authorization design, and more practical Laravel judgment.
